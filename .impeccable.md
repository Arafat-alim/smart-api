## Design Context

### Users
Backend developers, API teams, startup/hackathon builders, DevOps/platform engineers, open-source API authors. They land on this page mid-workflow (already have a Postman collection in hand) looking to skip hours of manual doc/SDK/mock-server writing. Technical audience — comfortable with jargon (SDK, OpenAPI, cURL), skeptical of hype, quick to bounce if the page feels like generic SaaS marketing.

### Brand Personality
Elegant, premium, modern, minimal, fast, developer-first, trustworthy, "local-first / no data leaves your browser." Confident and technical, not hypey. Reference points: Linear, Stripe, Vercel, Arc Browser — restrained, high-craft dev tool marketing, not startup-cliché.

### Aesthetic Direction
Dark theme (matches the existing app: near-black navy `#0f172a` base, slate surfaces `#1e293b`, slate borders `#334155`, indigo accent `#4f46e5`/`#818cf8`). A concept reference image exists at `landing-page/ChatGPT Image Jul 19, 2026, 07_06_24 PM.png` — accurate to the real product's actual capabilities (upload Postman collection → generate docs/SDK/types/OpenAPI/cURL/hooks/mock server, all client-processed) and useful for palette/section-shape/tone. Anti-references: startup-cliché gradient hero blobs, heavy drop shadows, generic AI-slop card grids.

### Design Principles
1. Truthful copy only — describe the real one-shot generator (upload → 7 artifacts → zip), never the "full API testing workspace" framing (Collections/Environment/History/Variables do not exist in this product).
2. Reuse the app's existing dark-slate/indigo palette — don't invent a new brand color.
3. Motion is purposeful: one well-orchestrated scroll story, not scattered micro-interactions everywhere.
4. No fabricated screenshots of features that don't exist; no fabricated named-person testimonials.
5. Left-aligned, asymmetric layouts over centered/card-grid monotony where it fits the section's content.
